# ADR-0004. Требовать подтверждение оператора для эксплуатационных действий

## Статус

Принято.

## Контекст

Система работает в контексте высокоскоростной железнодорожной магистрали. Ошибочная автоматическая реакция на ложную тревогу может привести к операционным последствиям. При этом ИИ-классификация в MVP еще требует накопления данных и проверки качества.

## Решение

В MVP система не выполняет эксплуатационные действия автоматически. Она может:

- создать кандидата события;
- классифицировать событие;
- оценить критичность;
- приложить видео;
- предложить оператору действие;
- создать задание путевой службе после подтверждения.

Подтверждение, отклонение и создание задания выполняет авторизованный оператор. Все действия пишутся в audit trail.

## Альтернативы

| Альтернатива | Почему не выбрана |
|---|---|
| Полностью автоматическая реакция | Слишком высокий риск для MVP и недостаточно подтвержденное качество моделей |
| Только информационные тревоги без заданий | Система хуже закрывает задачу реакции и эксплуатации |
| Автоматическая реакция только для низкого риска | Можно добавить позже, после статистики и регламентов |

## Последствия

Плюсы:

- снижается риск ошибочных автоматических действий;
- проще защищать архитектуру на первой итерации;
- решения оператора создают данные для улучшения модели;
- сохраняется полная трассируемость.

Минусы:

- время реакции зависит от оператора;
- нужна удобная очередь инцидентов и алерты о задержке;
- часть сценариев остается полуавтоматической.

## Когда пересмотреть

Решение можно пересмотреть после пилота, если качество классификации, регламенты и требования безопасности позволят автоматически выполнять часть низкорисковых действий.
