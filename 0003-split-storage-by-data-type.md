# ADR-0003. Разделить хранилища по типам данных

## Статус

Принято.

## Контекст

Система хранит разные типы данных:

- транзакционные данные инцидентов, заданий и аудита;
- геометрию участков и координаты;
- временные ряды признаков DAS;
- видео, акустические фрагменты и версии моделей.

Эти данные отличаются по объему, способу доступа, срокам хранения и требованиям к запросам.

## Решение

Использовать разные хранилища:

- PostgreSQL/PostGIS - источник истины по участкам, инцидентам, заданиям, аудиту и геоданным;
- TimescaleDB - временные ряды признаков, агрегаты и тренды;
- S3/MinIO - крупные бинарные артефакты: видео, DAS-фрагменты, модели.

## Альтернативы

| Альтернатива | Почему не выбрана |
|---|---|
| Хранить все в PostgreSQL | Упрощает разработку, но плохо подходит для крупных видео и акустических фрагментов |
| Хранить все в object storage | Не подходит для транзакций, статусов и геозапросов |
| Использовать только time-series DB | Не закрывает карточки инцидентов, роли, задания и audit trail |

## Последствия

Плюсы:

- каждое хранилище используется по назначению;
- проще управлять retention policy для тяжелых артефактов;
- PostGIS закрывает геопривязку и поиск ближайших объектов;
- TimescaleDB подходит для трендов цифрового двойника.

Минусы:

- больше инфраструктурных компонентов;
- нужны backup и monitoring для каждого слоя;
- разработчикам нужно учитывать согласованность metadata и артефактов.

## Когда пересмотреть

Решение стоит пересмотреть, если MVP будет запускаться в сильно ограниченной инфраструктуре и эксплуатационная сложность окажется важнее преимуществ разделения.
