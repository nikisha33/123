# ADR

Журнал архитектурных решений для курсовой работы "DAS Monitor ВСМ".

## Список решений

| ADR | Статус | Решение |
|---|---|---|
| [0001](0001-use-edge-central-architecture.md) | Принято | Использовать edge+central архитектуру |
| [0002](0002-use-kafka-for-event-streaming.md) | Принято | Использовать Kafka для потоковой обработки событий |
| [0003](0003-split-storage-by-data-type.md) | Принято | Разделить хранилища по типам данных |
| [0004](0004-require-operator-confirmation-for-actions.md) | Принято | Требовать подтверждение оператора для эксплуатационных действий |
| [0005](0005-make-digital-twin-a-separate-service.md) | Принято | Выделить цифровой двойник в отдельный сервис |

## Правила ведения

- Один ADR фиксирует одно архитектурное решение.
- ADR не заменяет раздел архитектуры, а объясняет компромисс.
- Если решение меняется, новый ADR должен явно заменить старый.
- В курсовой работе ADR считаются частью сдаваемой документации.
